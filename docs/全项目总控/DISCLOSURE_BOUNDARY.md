# Public Disclosure Boundary

## 1. 默认原则

公开仓库只发布可迁移的方法论资产。任何未明确进入公开包的资料，都默认视为私有资料。

| 等级 | 是否可外发 | 说明 |
|---|---|---|---|
| Public | 可以 | 已脱敏、可开源、可发客户 |
| Private | 不可以 | 私有策略、过程记录、未脱敏案例、本地素材 |
| Generated | 需复核 | 工具输出、验证证据、适配目录、缓存文件 |

没有明确标为公开的内容，不得默认外发。

## 2. 不能开源或外发的内容

- 私有策略材料
- 过程记录和未整理证据
- 未脱敏客户、项目、部署、账号或联系方式
- 本机绝对路径和个人环境配置
- 未脱敏案例
- 未审计的工具输出、截图、日志、缓存
- 可复用前还没有泛化的专属实现细节

## 3. 可公开内容

- 简化版理念说明
- 脱敏后的方法论总览
- 简化版 AGENTS 示例
- 少量基础能力单元示例
- 轻量 demo
- 不含私有策略的公开版评分表

## 4. 对外材料生成流程

1. 从私有材料提炼摘要。
2. 删除客户、项目、部署、账号、路径和策略动作。
3. 改写为公开语言。
4. 放入 `docs/公开材料/`。
5. 运行审计或人工复核。
6. 再用于开源、客户沟通或路演。

## 5. 当前结论

公开仓库只表达产品化方法论和可迁移能力，不发布私有策略、过程记录和未脱敏案例。
